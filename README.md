# 🚀  API Testing Collection Postman

## 📌 Project Overview

This repository contains multiple Postman collections designed for API testing across different scenarios such as API chaining, end-to-end testing, and domain-specific APIs.

The objective of this repository is to:

* Validate API functionality using Postman
* Perform API chaining and workflow testing
* Execute end-to-end API test scenarios
* Demonstrate hands-on experience in API testing

---

## 📂 Repository Contents

This repository includes the following Postman collections:

* 🔗 **API Chaining.postman_collection.json**
  → Demonstrates chaining of multiple API requests where output of one API is used as input for another

* 🛒 **E-commerce API.postman_collection.json**
  → Covers API testing scenarios for an e-commerce domain including cart and product-related operations

* 🔄 **E2E Testing.postman_collection.json**
  → End-to-end API testing flow covering complete business scenarios

* 📁 **My Collection1.postman_collection.json**
  → General API testing collection with multiple request examples

* 🎓 **Student API.postman_collection.json**
  → API testing scenarios for student-related CRUD operations

## 🚀 How to Use

1. Download any `.postman_collection.json` file
2. Open Postman
3. Click **Import**
4. Select the collection file
5. Run the requests individually or using Collection Runner
